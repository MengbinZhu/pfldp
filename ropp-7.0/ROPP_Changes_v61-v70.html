1. Introduction

This document summarizes the significant changes to an update of the Radio Occultation Processing Package (ROPP) Version v7.0 from the previous release, Version 6.1. The descriptions below are organized by module.
top

2. Generic

  • Added tropopause height diagnostic, in accordance with ROM SAF CDOP-2 plan.
  • Introduce "man" pages for all ROPP tools (in addition to those in ropp_io module).
  • Use of GRAS data in tests of the ropp_pp module in the Test Folder.
  • Use DMI operational sigma(N) (ie min = 0.02 N-units) in tests of the ropp_1dvar module in the Test Folder.
  • Revision of configuration script to allow building of ECMWF BUFR library with pgf95 v11.
  • Revision of configuration script to allow building of HDF5 and GRIB-API libraries with sunf95.
  • Additional configuration scripts to support building dependency libraries and ROPP modules with ifort XE-2013 (v13).
  • Minor improvements and bug fixes, in response to user requests.

3. ROPP_FM

top

4. ROPP_1DVAR

  • No significant changes.

top

5. ROPP_PP

  • Inclusion of tool to generate tropopause height (TPH) from all/any {bending angle, refractivity, dry temperature, background temperature} RO profile. The first two locate the `kink' in the profile at the tropopause by looking for the maximum of the covariance transform of the bending angle or refractivity; the last two search for a tropopause defined by the WMO lapse-rate criterion, or, alternatively, the minimum temperature in the lower part of the profile. 
  • Update of ropp_pp_invert_tool to cope better with operational M02 and M01 data.
  • Corrected sign in cl_exphase_p2 calculation in ropp_pp_grasrs2ropp.
  • Update of ropp_pp_spectra_tool to let it handle GRAS raw sampling data.

top

6. ROPP_IO

  • Update grib2bgrasc to allow radius of curvature and undulation to be input.
  • Copy level_type as part of lev2d ROPP substructure.
  • Improve lon_in=lon_out check in grib2bgrasc.
  • Quick fix to ropp_io_read_ncdf_get to allow ingestion of UCAR level 1a data (rescaling input in km and km/s).

top

7. ROPP_UTILS

  • Rearrangement of build in order to bypass some problems caused by the inclusion of USE typesizes in arrays/arrays.f90 reported at ROPP6.1.
